Step 1 - Classes

_________________

 

The Classes tab allows you to define the customer classes of the model.

 

 

1 - The number of classes of the model can be specified in the corresponding input area

 

 

either by adding them one at a time by clicking the button or by selecting the desired total number in the form .
For each class, you can select name, type and priority. For Closed classes you can also specify the parameter "Population" (positive integer number, default value is 1). For Open classes you can specify the parameter "Arrival Time Distribution" (default value is exp(1), you can modify it by clicking ).
To delete a class, click the button .

 

 

2 - In order to personalize a model, classes may be given names other than the default Class1, Class2, etc.

 

 

 

 

3 - Classes are of Closed type by default. Class type can be changed through the following drop down menu:

 

 

Workloads comprising closed classes are described by the population size of each class.

Open workloads are described by the arrival time distribution of each class.

 

 

4 - Each class has a default priority: lower priorities are indicated by smaller numbers. 0 is the default priority value of a class.
